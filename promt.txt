Ты — опытный фронтенд-разработчик. Твоя задача: с нуля создать браузерную мини-игру "Antifraud Desk: Сигнал".

ТРЕБОВАНИЯ:
Игра должна геймифицировать рабочий процесс специалиста антифрода. Пользователь видит рабочий кабинет с различными приборами и мониторами, запоминает скрытую последовательность событий и воспроизводит её. Должно быть понятно без подписей, что это и как играть.

СТРУКТУРА ПРОЕКТА:
Создай три файла: index.html, style.css, game.js в одной папке. Игра должна запускаться локально без сервера.

ГЕЙМПЛЕЙ ПОДРОБНО:
1. Стартовый экран: ввод имени игрока, выбор режима (Одиночный / Дуэль на одном ПК), кнопка "Начать".
2. Одиночный режим:
   - Начальный баланс: 1 000 000 ₽.
   - Уровень 1: длина последовательности 3. С каждым уровнем длина +1.
   - Фаза демонстрации: дважды показывается последовательность событий. Каждое событие — подсветка и анимация одного из 9 приборов (описаны ниже). Между истинными событиями вставляются N случайных ложных сигналов.
   - Фаза ввода: игрок кликает по приборам в нужном порядке. На экране отображается текущая введённая цепочка иконками. Есть таймер на ввод (7 + номер уровня секунд).
   - Если порядок верный: переход на следующий уровень, баланс не меняется.
   - Если ошибка: вычитается штраф = 10000 * уровень. При балансе <= 0 игра заканчивается.
   - Таблица рекордов: сохраняется в localStorage. Поля: Имя, Макс. раунд, Дата.
3. Дуэльный режим (hotseat):
   - Два игрока вводят имена. Ходят по очереди на одном и том же уровне. Сначала ходит Игрок 1, если проходит — ход переходит Игроку 2 на том же уровне но с другой последовательностью. Проигравший выбывает. Побеждает оставшийся.
   - Добавь кнопку "Подстава" (доступна раз за матч): перед демонстрацией противнику добавляет 2 лишних ложных сигнала.
4. Генерация последовательности:
   - Приборы: 'Красный монитор', 'Синий монитор', 'Зеленый монитор', 'факс', 'принтер', 'стационарный телефон', 'мобильный телефон', 'Ноутбук', 'планшет'.
   - Истинная последовательность генерируется случайно без повторов подряд.
   - Ложные события выбираются из оставшихся приборов.

ИНТЕРФЕЙС КАБИНЕТА (описать в CSS и JS):
- Игровое поле: нарисованный офисный кабинет где расположены интерактивные предметы который подают сигналы.
- При демонстрации: активный прибор получает класс 'active' (яркая тень, масштаб, возможно мерцание).
- В фазе ввода приборы кликабельны. При клике воспроизводится короткий звук (использовать базовый Audio или Web Audio API с простыми осцилляторами, чтобы не зависеть от файлов).
- Информационная панель: Баланс, Раунд, Таймер (прогресс-бар), Текущая последовательность (ряд мини-иконок).

ТЕХНИЧЕСКИЕ ДЕТАЛИ:
- Используй современный JavaScript (ES6+), HTML5, CSS3 (Flex/Grid).
- Состояние игры храни в объекте. Анимации делай через CSS transition и keyframes.
- Для генерации звука напиши простую функцию playSound(type) с Web Audio API.
- Адаптивный дизайн: минимальная ширина 800px, игра центрируется на странице.

ДОПОЛНИТЕЛЬНЫЕ ФИШКИ:
- Визуальная "стресс-шкала" — заполняется при неправильном клике, при полном заполнении провал раунда.
- Кнопка "Сброс ввода" для очистки цепочки.

Чек-лист перед отправкой:

- Звук работает только после клика на стартовом экране
- Нет ни одного alert() или текстового туториала
- Лидерборд сохраняется и читается корректно
- Дуэль корректно переключает состояние без утечек памяти
